# 功能说明

## 1. Model Hub（配置编辑）

功能点：
- 选择已配置方案（下拉）
- 编辑字段：Alias、Provider（只读展示）、Base URL、API Key、Model、Max Tokens、Enabled
- 测试连接
- 保存方案（支持 Alias 重命名）

交互规则：
- Alias 为空不可保存
- Alias 改名是“重命名语义”，不是复制
- 保存采用版本冲突控制，冲突时提示并刷新

## 2. Model Switch（切换与删除）

功能点：
- 展示所有方案（Alias/Provider/Model/Key 脱敏/状态）
- 勾选切换激活方案
- 删除指定方案

交互规则：
- 删除前二次确认
- 仅剩最后一个方案时禁止删除
- 若删除的是当前激活方案，自动切换到剩余首个方案

## 3. 配置文件双向同步

页面 -> 文件：
- 保存、切换、删除均通过后端落盘 TOML

文件 -> 页面：
- 后端监听 TOML 文件变更
- 前端通过 SSE 接收变更事件并刷新配置

## 4. 版本冲突保护

- 所有写操作要求传入 `base_version_hash`
- 当后端当前版本与提交版本不一致时返回冲突
- 前端冲突后触发刷新，避免覆盖他人/外部修改
