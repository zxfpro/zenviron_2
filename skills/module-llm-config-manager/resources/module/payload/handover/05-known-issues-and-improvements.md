# 已知问题与改进计划

## 已知问题

1. 前端端口不固定
- 现象：5173 被占用时自动切换，用户易混淆
- 影响：访问错误端口导致误判“页面加载失败”

2. 密钥明文存储
- 现状：`api_key` 明文写入 TOML
- 风险：本地泄露风险高

3. 前端测试覆盖不足
- 现状：仅有基础渲染测试
- 风险：UI 逻辑回归难以及时发现

4. 缺少权限与审计
- 现状：任何可访问服务的人可修改配置
- 风险：误操作或越权修改

## 改进计划

### P0（短期）

- 增加统一启动脚本并固定端口检测提示
- 增加前端关键交互测试（保存/删除/切换/冲突）
- 增加配置变更日志（至少记录 alias、动作、时间）

### P1（中期）

- 密钥安全：支持环境变量引用或加密存储
- 引入 RBAC（只读/编辑/管理员）
- 增加删除回收站或软删除机制

### P2（长期）

- 多环境配置（dev/test/prod）
- 批量导入导出方案
- 完整观测能力（metrics + tracing + alert）
