# PRD 0.0.1 - 看板任务管理系统 MVP

## 产品目标
- 基于 `code 2.html` 权威原型，交付可用的看板任务管理系统。
- 支持任务全生命周期：创建、编辑、流转、归档、恢复、删除。
- 支持多角色成员（管理员/产品/开发/测试）在无鉴权模式下的协作展示。

## 核心页面
- 看板页：多泳道任务展示、任务 CRUD、任务流转、筛选。
- 归档页：归档任务列表、恢复。
- 团队页：成员负载与最近活动。
- 设置页：工作区名称和主题配置。

## MVP 范围
- 不做登录鉴权，不做权限拦截。
- 单工作区模型。
- 预置成员、预置泳道、预置演示数据。
- 默认泳道包含：`Backlog/Ready/Waiting/Doing/Blocked/Done`。
- 新建任务通过弹窗表单提交，默认进入 `Backlog(预备池)`，支持优先级/负责人/预估时间（下拉）配置。
- 看板卡片支持拖拽跨泳道移动与泳道内排序。

## 验收标准
- API 可完整支持前端四页面功能。
- 前端可以通过真实 API 操作任务并实时刷新状态。
- 单元测试和集成路径测试可运行并通过。
