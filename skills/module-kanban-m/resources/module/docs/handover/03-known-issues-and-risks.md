# 已知问题与风险

## 当前已知问题

1. 拖拽排序持久化能力有限
- 表现：当前主要依赖前端排序体验，后端未单独维护稳定的卡片排序字段。
- 影响：复杂并发编辑时，排序一致性可能弱于专业看板工具。

2. 前端代码集中在 `App.tsx`
- 表现：页面、组件、状态逻辑耦合偏高。
- 影响：后续新增功能时维护成本上升。

3. 缺少端到端自动化测试
- 表现：目前以后端接口测试为主。
- 影响：UI 回归问题发现滞后。

4. 未接入鉴权与权限
- 表现：无登录体系，角色仅标签化。
- 影响：不适用于多租户或对权限敏感场景。

## 工程风险

1. 运行环境依赖网络
- Docker 首次构建需要拉取镜像和依赖，弱网可能导致构建失败。

2. 配置漂移风险
- 本地 SQLite 与 MySQL 语义差异可能导致边界行为不同。

3. 文档与代码不同步风险
- 未来迭代需在每次版本更新同步更新 `CHANGELOG` 和 handover 文档。
